Network Requirements

    We need:

  • Unlimited workload mobility - consolidate and split workload requirements across DC
  • VM Mobility

    • This means we use:

  • VLAN sprawl
  • Layer 2 everywhere

Issues:

  1. The above items result in large failure domains
  2. Spanning-tree in the fabric
  3. VLAN scalability

Virtualisation Requirements

  • Virtual connectivity defined by business requirements (internal addressing, services and security)
  • Isolate segments
  • Virtual QoS

Configuration Requirements

  • Centralised configuration
  • Single Point of Management
  • Consistent, network wide, policy-based configurations

Automation Requirements

  • Centralised configuration
  • Visibility of network and higher abstractions
  • Customer portal: API

DFA

Dynamic Fabric Automation delivers to all of these requirements

Network

  • Any subnet, anywhere, rapidly
  • Failure domain size reduced
  • Seamless L2 / L3 communication across fabric
  • Vast scale
  • Resilient
  • Topologically flexible

Virtual Fabrics

  • Any workload, any vFabric, anywhere
  • Scalable and secure
  • vFabric tenant visibility
  • Segmentation at L2/L3

Configuration

  • Automated Network Provisioning
  • Common Point of Fabric Access
  • Network, vFabric and Host Visibility

Workload automation

  • Open standard interfaces (REST, XMPP)
